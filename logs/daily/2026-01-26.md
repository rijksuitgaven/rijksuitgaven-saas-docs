### 18. Entity Resolution: Recipient Name Normalization

**Problem:** Same entity appeared multiple times with different spellings:
- "N.V. Nederlandse Spoorwegen" vs "Nederlandse Spoorwegen" vs "NEDERLANDSE SPOORWEGEN N.V."
- "NS Vastgoed B.V." vs "NS Vastgoed BV"
- "ProRail B.V." vs "Prorail BV"

**Solution:** Created `normalize_recipient()` function that handles:
- Casing variations (UPPER)
- B.V./BV variations (stripped)
- N.V./NV variations (stripped)
- Extra spaces (normalized)

**Result:**

| Metric | Before | After |
|--------|--------|-------|
| Total entities | 466,827 | 451,445 |
| **Duplicates merged** | - | **15,382 (3.3%)** |

**Examples:**
- Nederlandse Spoorwegen: 3 rows → 1 row (€90M combined)
- NS Vastgoed: 2 rows → 1 row (€22M combined)
- ProRail: multiple → 1 row (€12.7B combined)

### 20. Fixed Integer Overflow in Typesense Sync Script

**Problem:** `psycopg2.errors.NumericValueOutOfRange: integer out of range`

**Cause:** `bedrag * 1000` overflowed PostgreSQL integer limit on large amounts.

**Fix:** Cast to BIGINT before multiplication:
```sql
COALESCE(bedrag, 0)::bigint * 1000 as bedrag
```
